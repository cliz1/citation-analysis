Abstract. In this paper, we introduce the subÔ¨Āeld bilinear collision
problem and use it to construct an identiÔ¨Ācation protocol and a signature
scheme. This construction is based on the MPC-in-the-head paradigm
and uses the Fiat-Shamir transformation to obtain a signature.
1
Introduction
The use of the Multi-Party Computation in the Head (MPCitH) paradigm for
Zero-Knowledge (ZK) protocols was introduced in [23]. The general idea is to use
any NP-relation R(x, w) to obtain a ZK-protocol in which a prover P convinces a
veriÔ¨Āer V that she knows a valid witness w for a given (public) value of x without
revealing any information about x. In this paper, we introduce a problem which
naturally arises by using the standard techniques for Ô¨Ānding discrete logarithms
in small characteristic Ô¨Ānite Ô¨Āelds to construct such a protocol. We then use the
Fiat-Shamir heuristic [16] to turn the corresponding MPCitH protocol into a
signature scheme. The advantage of introducing this new problem is that it can
easily be transformed into an MPC protocol which yields a smaller signature
size compared to existing post-quantum schemes.
We deÔ¨Āne the problem that we consider throughout this paper as follows:
The SubÔ¨Āeld Bilinear Collision (SBC) Problem. This problem depends
on three parameters: A prime power q and two positive integers k, n.
